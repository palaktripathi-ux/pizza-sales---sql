# pizza-sales---sql
🍕 Pizza Sales SQL Analysis Project
📌 Project Overview

This project focuses on analyzing a pizza restaurant's sales data using SQL. The objective is to extract meaningful business insights from transactional data by solving 13 SQL problems ranging from basic to advanced difficulty levels.

Through this project, I practiced essential SQL concepts such as Joins, Aggregations, Subqueries, Common Table Expressions (CTEs), Window Functions, Grouping, and Business KPI Analysis.

🎯 Project Objectives
Analyze sales performance and revenue generation.
Identify customer ordering patterns.
Discover best-selling pizzas and categories.
Evaluate peak business hours.
Generate actionable insights from raw sales data.

   
   📊 SQL Problems Solved
   
BASIC LEVEL

1.Retrieve the total number of orders placed.

2.Calculate the total revenue generated from pizza sales.

3.Identify the highest-priced pizza.

4.Identify the most common pizza size ordered.

5.List the top 5 most ordered pizza types along with their quantities.


INTERMEDEATE LEVEL

1.Find the total quantity ordered for each pizza category.

2.Determine the distribution of orders by hour of the day.

3.Find the category-wise distribution of pizzas.

4.Calculate the average number of pizzas ordered per day.

5.Determine the top 3 pizza types generating the highest revenue.


ADVANCED LEVEL

1.Calculate the percentage contribution of each pizza type to total revenue.

2.Analyze cumulative revenue generated over time.

3.Determine the top 3 revenue-generating pizza types within each category.
